# 云服务器部署手册

> 目标：把 ETH 交易系统迁移到云服务器，实现 24 小时不间断运行。

---

## 第一步：购买服务器

### 推荐配置（够用即可，省钱优先）

| 项目 | 推荐 |
|------|------|
| 云厂商 | 阿里云轻量 / 腾讯云轻量 / 雨云（便宜） |
| 系统 | Ubuntu 22.04 LTS（64位） |
| CPU | 1 核 |
| 内存 | 1 GB |
| 带宽 | 1 Mbps 以上 |
| 地区 | **香港 / 新加坡**（直连 OKX，无需代理） |
| 费用 | 约 ¥30-60/月 |

> ⚠️ **关键**：选香港或新加坡节点，能直接访问 OKX，不需要代理！
> 大陆节点会被墙，无法连接 OKX。

---

## 第二步：连接服务器

购买后会得到：服务器 IP、root 密码

```bash
# 在你的 Mac 终端执行（替换 IP）
ssh root@服务器IP
```

输入密码，成功进入服务器终端。

---

## 第三步：传输 API 密钥（重要！密钥不能上 GitHub）

在你的 **Mac 本地终端** 执行（不是服务器）：

```bash
# 把 .env 文件安全传到服务器
scp /Users/gaofeng/Documents/okx_eth_bot/.env root@服务器IP:/tmp/okx.env
```

---

## 第四步：一键部署

在服务器终端执行：

```bash
# 下载并运行部署脚本
curl -fsSL https://raw.githubusercontent.com/gaofeng124/okx_eth_bot/main/deploy/setup_server.sh | bash
```

脚本会自动完成：
- 安装 Python 3、git 等依赖
- 克隆代码仓库
- 创建虚拟环境，安装依赖包
- 自动清除代理设置（云服务器直连 OKX）
- 配置 systemd 开机自启动
- 启动守门人进程

部署完成后，把 .env 移动到正确位置：

```bash
mv /tmp/okx.env /root/okx_eth_bot/.env
# 自动修复代理（留空，云服务器不需要）
sed -i 's|OKX_WS_PROXY=.*|OKX_WS_PROXY=|g' /root/okx_eth_bot/.env
# 重启服务
systemctl restart okx-watchdog
```

---

## 第五步：验证运行

```bash
# 查看守门人 + 交易系统是否在运行
systemctl status okx-watchdog

# 查看实时日志
tail -f /root/okx_eth_bot/data/logs/watchdog.log

# 查看完整状态报告
cd /root/okx_eth_bot && .venv/bin/python status.py
```

```bash
# 重启系统
systemctl restart okx-watchdog

# 实时日志
tail -f /root/okx_eth_bot/data/logs/watchdog.log

cd /root/okx_eth_bot && .venv/bin/python status.py
```

## 自动升级流程（部署后自动生效）

```
远程 Agent（每小时）
    → 分析行情 + 日志
    → 修改 settings.py
    → 推送到 GitHub
        → 守门人检测到新提交（5分钟内）
        → git pull 拉取新代码
        → 自动重启交易系统
```

云服务器 24 小时在线，这个循环永不停止。

---

## 与本地的区别

| 项目 | 本地 Mac | 云服务器 |
|------|---------|---------|
| `OKX_WS_PROXY` | `http://127.0.0.1:7897` | **留空**（直连） |
| 运行时间 | 开机才运行 | 24/7 不间断 |
| 断电重启 | 需手动重启 | systemd 自动重启 |
| 开机自启 | 无 | systemd enable 已配置 |

---

## 安全建议

1. 服务器只开放 SSH 端口（22），其他端口关闭
2. 改 SSH 端口（可选，默认 22）
3. `.env` 文件权限设置为 600：`chmod 600 /root/okx_eth_bot/.env`
4. 定期检查 API 密钥是否有异常调用
